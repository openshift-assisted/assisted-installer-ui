> **Reviewer summary:** This fix adds the `autoconf: false` field to IPv6 static network configurations in the nmstate YAML generator. The changes are in `libs/ui-lib/lib/ocm/components/clusterConfiguration/staticIp/data/` and include type updates, the YAML generation logic, and comprehensive unit tests.

## What

Add missing `autoconf: false` field to IPv6 static network configurations in the UI form view.

## Why

When creating a dual-stack static network configuration with IPv6, the UI was not including the `autoconf: false` field in the generated nmstate YAML. According to nmstate specifications, the `autoconf` field should be set to `false` when using static IPv6 addresses (it's only applicable to IPv6, not IPv4). Without this field, static IPv6 configuration may be ignored by the network manager.

## How

1. Updated the `NmstateProtocolConfig` type in `nmstateTypes.ts` to include an optional `autoconf` field
2. Modified `getNmstateProtocolConfig()` function in `nmstateYaml.ts` to:
   - Accept a `protocolVersion` parameter
   - Set `autoconf: false` when the protocol version is IPv6
3. Updated all callers of `getNmstateProtocolConfig()` to pass the protocol version:
   - `formDataToInfraEnvField.ts` (2 call sites)
   - `dummyData.ts` (1 call site)
4. Added comprehensive unit tests in `nmstateYaml.test.ts` to verify:
   - IPv4 configs do not include the autoconf field
   - IPv6 configs include `autoconf: false`
   - Dual-stack interface generation includes autoconf in the YAML output
