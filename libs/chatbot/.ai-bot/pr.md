> **Reviewer summary:** This PR fixes the static IPv6 network configuration form to include the required `autoconf: false` field in the generated nmstate YAML. The changes are in `libs/ui-lib/lib/ocm/components/clusterConfiguration/staticIp/data/` and `libs/chatbot/package.json` (build fix).

## What

Add `autoconf: false` field to IPv6 static network configurations generated by the UI form view.

## Why

When creating dual-stack or IPv6 static network configurations through the UI form view, the generated nmstate YAML was missing the `autoconf: false` field for IPv6 interfaces. This field is required for static IPv6 configurations to prevent SLAAC (Stateless Address Autoconfiguration) from overriding the static IP address. Without it, the static IP configuration could be ignored by the network stack.

## How

1. **Updated nmstate type definitions** (`nmstateTypes.ts`):
   - Added optional `autoconf?: boolean` field to `NmstateProtocolConfig` type

2. **Modified protocol config generation** (`nmstateYaml.ts`):
   - Updated `getNmstateProtocolConfig()` to accept `protocolVersion` parameter
   - Set `autoconf: false` when `protocolVersion === ProtocolVersion.ipv6`
   - IPv4 configs do not include the autoconf field (as it's IPv6-only)

3. **Updated all call sites**:
   - Modified `formDataToInfraEnvField.ts` to pass protocol version to `getNmstateProtocolConfig()`
   - Updated `dummyData.ts` to pass protocol version

4. **Added comprehensive tests** (`nmstateYaml.test.ts`):
   - Test that IPv4 configs exclude the autoconf field
   - Test that IPv6 configs include `autoconf: false`
   - Test dual-stack interface generation

5. **Fixed chatbot build issue** (`libs/chatbot/package.json`):
   - Fixed rsync commands to copy assets to correct directory structure
